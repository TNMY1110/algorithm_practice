# 수업 목표
- 자료구조 (스택, 큐)
- 그래프 표현
- BFS / DFS

---

# 용어 정리
- 알고리즘 : 문제를 푸는 순서를 적어둔 레시피. 단계적 절차
- 자료구조 : 데이터를 효율적으로 담는 그릇의 종류. 스택, 큐, 그래프 등이 각각 다른 모양의 그릇
- 스택(stack) : 접시를 쌓는 것처럼 마지막에 올린 것을 먼저 꺼내는 구조, LIFO(Last In, First Out)
- 큐(Queue) : 줄을 선 것 처럼 먼저 온 사람이 먼저 꺼내지는 구조, FIFO(First In, First Out)
- 그래프(Graph) : 지하철 노선도처럼 점(노드)과 선(간선)으로 연결 관계를 표현한 것
- 노드(Node) :그래프에서 동그라미로 표현되는 정점
- 간선(Edge) : 노드와 노드를 잇는 선
- BFS : 너비 우선 탐색. 물결이 퍼지듯 가까운 곳부터 차례로 탐색하는 방법
- DFS : 깊이 우선 탐색. 한 우물만 판다는 느낌으로, 갈 수 있는 끝까지 일단 가보고 막히면 되돌아 옴. (스택이나 재귀를 사용)
- 재귀(Recursion) : 마트료시카. 자기 자신을 다시 호출하여 문제를 해결하는 방식
- 방문(visited) : 길을 잃지 않기 위해 내가 이미 지나온 길에 체크 표시를 하는 것

---
# 선형 자료구조 (Linear Data Structure)
| 구분 | 설명 | 원리 | 비유 |
| :--- | :--- | :--- | :--- |
| **스택 (Stack)** |  마지막에 올린 것을 먼저 꺼냄 | **LIFO** (Last In, First Out) | 접시 쌓기 | 
| **큐 (Queue)** |  먼저 온 것을 먼저 꺼냄 | **FIFO** (First In, First Out) | 줄 서기 | 
---


# 그래프 (Graph) 구조
| 구분 | 설명 | 비유 |
| :--- | :--- | :--- |
| **노드 (Node)**    | 데이터의 주체 | 지하철역, 도시, 로봇 위치 |
| **간선 (Edge)**    | 데이터 간의 연결 고리 | 지하철 노선, 도로 |
| **방문 (visited)** | 무한 루프 방지를 위해 탐색한 노드를 체크하는 행위 | 헨젤과 그레텔의 빵 |
